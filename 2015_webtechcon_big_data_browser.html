Motivation: Warum denn überhaupt Big Data im Browser?

  • Diagramme sind eine großartige Art, viele Daten in ein übersichtliches Format zu bringen
  • Erst dynamische, interaktive Diagramme ermöglichen die optimiale Nutzung von Daten
  • Damit dynamische Diagramme im Browser optimal funktionieren, braucht man verzögerungsfreie Interaktionen
  • Verzögerungsfreie Interaktionen sind nur durch lokalen Programmcode und lokale Daten möglich
  • Das kann die Verarbeitung von bis zu mehreren Millionen Datensätzen im Browser erfordern

D3.js: Dynamic graphics in the browser

  • http://d3js.org/
  • Daten werden in Darstellung transformiert
  • Transformationsvorschrift ist deklarativ
  • Grundsätzlich ist D3 sehr allgemein gehalten
  • Haupteinsatzgebiet sind aber Diagrammer mit SVG (Vektor-Grafiken im Browser)
  • Darstellungsdaten werden dabei an SVG-Elemente gebunden

D3 Beispiel-Code

var updateSelection = d3.select("body") // select the body tag
    .selectAll("p")  // select all p tags which are children
    .data([4, 8, 15, 16, 23, 42]) // bind elements with data
    .text(function (d) { // data gets passed in
        return "I’m number " + d + "!"; // result gets displayed
    })
    .style("font-size", function (d) {
        return 2 * d + "px";
    });

Crossfilter: Filtering millions of data sets in real time

  • http://square.github.io/crossfilter/
  • Kann im Browser Millionen von Datensätzen in Echtzeit filtern
  • Datensätze werden nach dem Laden indiziert
  • Kann mit D3 kombiniert werden
  • Von Crossfilter gefilterte Daten dienen dann als Daten für D3
